# 2015-On Using Very Large Target Vocabulary for Neural Machine Translation

> 2026.04.24

标题：On Using Very Large Target Vocabulary for Neural Machine Translation

年份： 2015年

作者：蒙特利尔大学

---



Jean et al. (2015) 提出了一种**基于重要性采样的大词表训练方法**，在不改变模型结构的情况下，允许 NMT 使用超大目标词表。

该方法属于**训练策略与工程优化**，而非新的网络架构。

这篇 2015 年的工作**不属于模型结构创新**，**没有提出新的网络架构**，而是一套**针对大词表 NMT 的训练与推理工程优化方法**，基于**重要性采样**与**候选列表解码**，在不改变模型结构的前提下，解决了传统 NMT 无法使用超大目标词表的计算瓶颈。



## 论文概述

#### 核心内容

该论文针对**神经机器翻译（Neural Machine Translation, NMT）** 中**目标词汇量过大导致计算复杂度高**的问题，通过三点方法彻底解决了神经机器翻译的大词表瓶颈：1）提出了一种基于**重要性采样（Importance Sampling）** 的近似训练方法，使得模型可以在训练时使用**超大目标词汇表**而不显著增加计算复杂度。2）并在解码阶段使用**动态候选词列表**保持高速推理。3）同时利用**注意力对齐**处理未登录词



#### 工作影响

用「重要性采样 + 解码候选列表」两套机制，同时解决训练与推理的大词表计算爆炸问题：

1. **训练时**：不限制词表大小，用**重要性采样**只算一小部分词的梯度，让训练复杂度**不随词表增大而上升**。
2. **推理时**：用**动态候选词列表**代替全词表 Softmax，让解码速度回到小词表水平。
3. 

该工作是NMT领域早期**解决大词汇表问题的重要尝试**，为后续如**层次softmax、采样策略、子词分割（BPE）** 等方法提供了启发，推动了NMT在实际系统中的应用。



#### 主要贡献

1. **问题识别**：
   - 传统NMT模型（如RNNsearch、LSTM）通常只使用3万~8万的高频词（shortlist），其余词被标记为 `[UNK]`（未知词），导致翻译性能下降。
2. **提出方法**：
   - 使用**重要性采样**来近似计算softmax归一化项，每次更新只使用一个小子集（如1.5万~5万词）来估计梯度，从而大幅降低计算量。
3. **解码优化**：
   - 推理时使用**候选词列表（candidate list）**，仅对每个源词最可能的目标词进行采样，进一步加速解码。
4. **实验结果**：
   - 在 **WMT'14 英法** 和 **英德** 翻译任务上，使用大词汇表的模型（RNNsearch-LV）在BLEU分数上**优于或匹配**之前的SOTA模型（包括Google的NMT和短语式SMT系统）。
   - 通过模型集成（ensemble），进一步提升了性能，接近甚至超过当时的最佳系统。
   - 大词表模型翻译质量显著提升，速度几乎不损失。

## 核心方案

#### 1. 训练阶段

#### 用「重要性采样」解决大词表训练爆炸

1. 原来的问题

标准 NMT 每次算概率都要对**全词表做 Softmax**：

- 复杂度 **O(|V|)**
- 词表越大越慢、显存爆炸

2. 论文怎么做（重要性采样）

- 每次更新梯度时，**只采样一小部分词 V′** 参与计算
- 用重要性权重近似原本需要全词表的期望梯度
- 让**训练复杂度固定**，和总词表 |V| 无关

3. 具体实现

- 把训练语料分块
- 每块预先构建一个**小词表子集 V′**（如 15k/30k/50k）
- 训练时只在 V′ 内算概率、算梯度
- 总词表可以是 **50 万**，但训练速度和 3 万词表一样快

4. 关键好处

- **支持超大词表（50 万词）**
- **训练速度不变**
- **显存占用不变**



#### 2. 推理（解码）阶段

#### 用「候选列表」加速大词表解码

1. 问题

训练完大词表模型后，如果推理时仍算全词表 Softmax，会**极慢**。

2. 论文方法：**Candidate List（候选词列表）**

每一步解码只算一个**小子集**，而不是全词表。

构建方式：

1. 取**高频 K 词**（如 3 万）
2. 用**双语词典**，为每个源词附加最多 K′ 个可能目标词
3. 合并成一个**动态小候选集**，只在这个集合上算 Softmax

3. 效果（论文 Table 3）

- 全词表大模型：GPU 每词 0.25s
- **候选列表 + 大词表**：GPU 每词 **0.05s**
- 速度几乎回到**传统小词表模型**水平！



#### 3. OOV 未登录词问题

用注意力对齐自动修复

论文 3.3 节解决大词表仍会出现的 `<UNK>` 问题：

- 模型生成 UNK 时
- 用**注意力权重 α_t** 找到它最对齐的源词
- 直接用词典或音译替换
- 大词表模型本身生成 UNK 更少，效果更强



#### 4. 整套方法完整流程

1. **词表**：使用**超大目标词表（50 万）**，不截断

2. **训练**：

   - 用**重要性采样**
   - 每次只在小子集（15k/30k/50k）上计算
   - 复杂度不随总词表增大而上升

3. **解码**：

   - 用**动态候选列表**（高频词 + 词典匹配词）
   - 速度回到小词表级别

4. **UNK 处理**：

   - 注意力对齐 → 替换源词翻译
   - 大词表本身更少出现 UNK

#### 三、为什么没有独立开源实现？

1. **时代背景限制**：2015 年的 NMT 论文大多不强制要求开源代码，这篇论文的实现细节偏工程化，依赖当时的 GPU 环境和内部工具链；
2. **后续技术迭代**：后续的技术（如 BPE/Subword 分词、混合精度训练、分层 Softmax 改进版）逐渐替代了这种 “分块词表” 的方案，论文的核心优化被整合进了更通用的框架中；
3. **工程实现成本高**：论文的分块词表、动态提议分布等细节实现较为繁琐，没有独立开源的价值。

------

#### 四、替代方案建议

如果你现在需要处理大词表 NMT 问题，更推荐使用：

1. **Subword 分词（BPE/SentencePiece）**：从源头减少词表大小，避免 OOV 问题；
2. **分层 Softmax 或负采样**：直接使用主流框架中的现成实现；
3. **候选列表解码（Shortlist）**：解码时使用高频词子集，大幅降低推理复杂度。

## 这篇论文，在训练阶段，和Sampled softmax有什么区别？

我直接**对照论文原文**，只讲**训练阶段**，把本文方法（Jean 2015）和 Sampled Softmax 的核心区别讲透，不掺推理、不绕弯子。

#### 核心结论（一句话）

- **Sampled Softmax**：训练时**随机采样**负样本，用局部集合近似全词表 Softmax，**有偏、随机、不利用语义**。
- **本文方法**：训练时用**重要性采样 + 语料分区动态词表**，近似**原始梯度的期望**，**非随机、上下文相关、更接近全词表梯度**。

------

#### 1. 近似对象完全不同（最本质区别）

Sampled Softmax

直接近似 **概率本身**：只在「真实词 + 随机采样负词」上算 Softmax，**完全抛弃全词表归一化**。

本文方法（论文 3.1 节）

近似的是 **log p (y) 的梯度**，把梯度拆成：∇log p = 正项 − **期望项**然后用重要性采样去**近似这个期望**，**保留原始梯度结构**。

→ 本文是**梯度层面近似**，Sampled Softmax 是**概率层面近似**。

------

#### 2. 负样本从哪来？（完全不同）

Sampled Softmax

- 从**全局词表随机采样**
- 服从**词频分布**（unigram）
- **和当前句子、源词、上下文无关**
- 纯随机，不看语义

本文方法（论文 3.1）

- 不随机采样
- 按**语料分块**预先构建动态子集 V'i
- 每个块只用**该块出现的词**
- 负样本是**当前句子相关的词**
- 是**语义 / 上下文感知**的候选集

→ 本文**不是随机采样**，Sampled Softmax 是**纯随机**。

------

#### 3. 数学近似方式不同

Sampled Softmax

用**局部小集合**代替全词表做 Softmax：p ≈ exp (...) / Σ_{采样集} exp (...)**有偏，偏离真实分布**。

本文方法

用**重要性采样（importance sampling）** 近似梯度里的**期望项**：E_P [∇ε] ≈ 加权和公式严格来自论文 **(8)(9)(10)**。

- 保留原始梯度形式
- 只是用小集合近似期望
- **偏差更小**

------

#### 4. 词表使用策略不同

Sampled Softmax

- 每步只更新：**1 个正例 + N 个随机负例**
- 其他词**完全不更新**
- 低频词几乎学不到

本文方法

- 按**语料块**固定一个动态词表 V'i
- 每步更新**正例 + 整个 V'i**
- 低频词在自己的语料块里能被充分训练
- 大词表信息保留更完整

------

#### 5. 训练偏差与效果

Sampled Softmax

- 随机采样导致**分布偏移**
- 低频词、专有名词效果差
- 训练快，但**模型质量妥协大**

本文方法

- 重要性采样更接近**全词表梯度**
- 动态词表让**低频词能学到**
- 训练同样快，但**模型质量更接近全词表**
- 论文实验证明：大词表明显涨 BLEU

------

#### 6. 最精简对比表（只看训练）

| 项目               | Sampled Softmax  | 本文方法（Jean 2015） |
| ------------------ | ---------------- | --------------------- |
| 近似目标           | 近似**概率**     | 近似**梯度的期望**    |

## 本文是**梯度层面近似**，Sampled Softmax 是**概率层面近似**。 是什么原因造成了两者各自的选择？

我直接给你**最本质、最根源、最符合论文动机**的答案，一句话先点破：

一句话根本原因

**本文想保留全词表训练的正确性；Sampled Softmax 只想最快降低训练成本。**所以一个选**梯度近似**，一个选**概率近似**。

------

#### 一、为什么本文选择「梯度层面近似」？

（完全来自论文 3.1 节的动机）

1. 作者的目标：**用超大词表，但不破坏模型学习**

论文要解决的是：**大词表训练太慢，但不能因为近似而学坏语义、低频词、对齐关系。**

2. 数学上：只有梯度能 “保留全词表期望”

标准 log p (y) 的梯度是：∇ log p = ∇ε(y) − E_p [∇ε(y)]

这个梯度**天然是全词表期望**，作者不想破坏它。所以作者选择：**只近似期望项，不改变梯度结构**

→ 这样训练出来的词向量**依然能和隐状态正确对齐**，推理时才能放心裁剪词表。

3. 工程上：作者想支持 “超大词表 + 低频词”

- 梯度近似 = 保留全局分布信息
- 低频词不会被随机采样漏掉
- 词向量空间更健康

**一句话：作者要 “正确”，所以选梯度近似。**

------

#### 二、为什么 Sampled Softmax 选择「概率层面近似」？

（来自 Mnih 2013 等原始动机）

1. 它的目标只有一个：**最快、最简单降低计算量**

它不管全词表分布，不管低频词，不管对齐，**只要训练能跑起来就行**。

2. 数学上：概率近似最简单、最好实现

直接把：p_full (y) → p_sampled (y)

在小集合上重新做 softmax，**不用推导梯度、不用期望、不用重要性采样**。

3. 工程上：随机采样最容易写代码

- 随机抽几个负样本
- 局部 softmax
- 不用分区、不用词典、不用对齐
- 完全不关心上下文

**一句话：它要 “简单快”，所以选概率近似。**

------

#### 三、最核心的选择原因对比（一眼看懂）

| 选择原因               | 本文（梯度近似）               | Sampled Softmax（概率近似） |
| ---------------------- | ------------------------------ | --------------------------- |
| **核心目标**           | 保留全词表训练正确性           | 最小化计算、最快训练        |
| **是否在乎分布**       | 在乎，必须接近真实 p (y)       | 不在乎，只要能收敛          |
| **是否在乎低频词**     | 非常在乎                       | 不在乎，随机采样随缘        |
| **是否为推理加速服务** | 是，训练为推理裁剪做准备       | 否，只解决训练慢            |
| **数学难度**           | 高（梯度 + 期望 + 重要性采样） | 低（局部 softmax）          |
| **近似代价**           | 小，几乎不破坏语义             | 大，破坏全局分布            |

------

#### 四、终极总结（最精炼）

- **本文选梯度近似**：因为要**超大词表、正确语义、对齐关系、低频词、推理可裁剪**，所以必须保留梯度的期望结构，只近似期望。
- **Sampled Softmax 选概率近似**：因为只想要**最简单、最快、最容易实现**的训练加速，不在乎分布偏差、不在乎低频、不在乎推理。
