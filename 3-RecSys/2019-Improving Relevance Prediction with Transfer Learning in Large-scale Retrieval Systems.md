# 2019-Improving Relevance Prediction with Transfer Learning in Large-scale Retrieval Systems

> 2026.05.14



场景：谷歌APP应用商店。

关键词：召回、显式相关性建模、相关性得分预测、MSE回归、迁移学习、双塔模型、隐式行为预测、大规模Softmax多分类



## 本文场景

工业界常规召回模型通过 点击正样本 + 全局随机负采样 优化点击相对概率，而本文模型以 人工显式相关性回归为主任务、隐式行为 Softmax 为辅助任务 ，直接学习真实相关性而非点击行为。

1. 工业常规召回模型

- 任务： 隐式行为匹配 ，不是相关性预测
- 标签：正样本：（query, item） 有点击 ；负样本： 全局随机采样未点击物品 
- 损失： 二元交叉熵 / 负采样 Softmax（NCE） 
- 目标：最大化 “点击样本” 相对 “随机负样本” 的排序概率
- 学到的： 用户兴趣 + 行为匹配模式 ， 不是人工定义的相关性 。是「点击相对概率」，不是「绝对相关性」

2. 本文模型

- 任务：显式「相关性回归」。主任务 相关性回归任务 ，标签是人工显式相关性分数；辅助任务：全局 Softmax（全库当负样本）
- 目标： 拟合真实相关性（ground truth） ，不是点击概率
- serving：只用相关性回归头 ， 不用点击预测头

> 3.2. Training with User Feedback
>
> When training a machine learning based retrieval system, the ideal way is to use users’ explicit feedback which reflects the relevance of an item to a query. However, asking for users’ explicit feedback is costly; hence, many existing systems use implicit feedback from user logs, such as clicks.
>
> In this paper, we study retrieval systems with both explicit and implicit feedback, where implicit feedback is abundant and explicit feedback is relatively sparse.

3. 为什么工业界主流召回建模「预测点击相对概率」、论文建模「预测相关性得分」？

- 工业界： 没人工相关性标签、成本极高 ，只能用点击当弱信号， 求快、求量、求点击率 
- 论文（Google）： 有预算做大规模人工标注 ，目标是 对齐人工相关性标准、提升长期用户满意度 ，解决 点击偏差 问题



## 核心创新点

1. 提出迁移学习框架：利用海量隐式反馈（点击、互动）辅助稀疏显式反馈（人工标注）训练，解决标注稀缺、昂贵问题
2. 双塔+共享底（Shared-Bottom）模型结构：双塔（Query塔/候选塔）基础上，底层共享、上层双分支，同时建模显式相关性与隐式用户行为
3. 两阶段训练策略：先训隐式反馈辅助任务（交叉熵），再微调显式主任务（MSE）；共享层用梯度停止，避免稀疏标注过拟合
4. 首次在工业级大规模检索系统落地：验证隐式反馈可正则化模型、提升泛化，显著降低相关性预测RMSE，支撑亿级候选检索
5. 统一多模态特征融合：协同信息+内容信息联合建模，比单一特征效果更强，适配检索多模态特征场景

一句话概括： 首次把双塔检索模型 和 迁移/多任务学习深度结合，用海量隐式反馈解决显式标注稀缺、易过拟合的工业痛点，并在大规模检索系统验证显著效果 。

下面分点说得更清晰：

1.  迁移学习思路：用“便宜的隐式反馈”救“昂贵的显式标注” 
   - 现实痛点：显式相关性标注（人工打分） 少、贵、质量不稳定 ；隐式反馈（点击、停留、互动） 海量、免费、易获取 。
   - 创先：把隐式反馈做成 辅助任务 ，先学用户行为，再迁移到显式相关性预测，显著提升稀疏标注下的模型效果。

2.  新模型结构：双塔 + 共享底（Shared-Bottom）双分支 
   - 基线：工业常用 双塔DNN （Query塔/候选塔独立编码，内积算相关性）。
   - 创先：底层 共享特征提取层 ，上层分出两个独立分支：
     - 一支： 隐式反馈塔 （建模点击/互动，softmax+交叉熵）；
     - 一支： 显式相关性塔 （建模人工标注，MSE回归）。
   - 特点：共享层学通用表示，上层分支适配不同任务，兼顾效率与效果。

3.  两阶段训练+梯度停止：防止稀疏标注过拟合 
   - 阶段1：先用 海量隐式数据 预训练共享层+隐式分支；
   - 阶段2：用少量显式数据 微调显式分支， 共享层梯度停止（不更新）。
   - 创先：避免小数据把共享层“带偏”，隐式数据天然起到 正则化 作用，模型越大效果越好（纯显式训练会过拟合、效果变差）。

4.  工业级落地验证：效果提升明确、结论扎实 
   - 场景：Google 大规模推荐检索（百万级候选、亿级隐式日志、十万级显式标注）；
   - 指标：相关性预测 RMSE 从  0.3464（纯显式）  降到  0.2673（显式+隐式迁移） ，降幅非常显著；
   - 结论：隐式反馈能有效正则化、提升泛化，解决工业检索最核心的“标注稀疏+过拟合”难题

5.  多模态特征统一建模：协同+内容信息联合优化 
   - 同时用协同过滤特征（用户-物品交互）+内容特征（文本/属性），比单用一类特征效果更强




## Transfer Learning体现

一句话总结：该方法通过 “源任务（海量隐式行为）→共享表示→目标任务（稀疏显式相关性）” 的范式，用 Shared-Bottom 结构和两阶段训练，把海量隐式数据学到的通用特征迁移给稀疏相关性任务，显著提升效果并防过拟合，是典型的迁移学习应用。

1. 知识从 “海量隐式行为反馈” 迁移到 “稀疏显式相关性任务”

  - 源任务（Source）：用户隐式行为预测（点击、互动），数据量极大、免费、噪声低；
  - 目标任务（Target）：人工标注相关性预测，数据极稀疏、昂贵、难标注；
  - 迁移核心：用源任务学到的 Query/Item 通用表示，直接帮助目标任务，提升稀疏数据下的效果，这就是典型的迁移学习（Transfer Learning）。

2. 模型结构：双塔 + Shared-Bottom 共享底层，实现参数迁移

  - 共享底层（Shared Bottom）：Query 塔、Item 塔的底层 Embedding 和隐藏层完全共享；
  - 上层双分支：
    - 隐式分支：学用户行为（源任务）
    - 显式分支：学相关性（目标任务）
  - 迁移逻辑：共享层学到的通用特征，被目标任务直接复用；相当于把源任务的知识 “迁移” 给目标任务

3. 训练方式：两阶段训练 = 预训练 + 微调（经典迁移范式）

  - 阶段 1：预训练（用隐式数据）

    先用数十亿隐式数据训练共享层 + 隐式分支，学到高质量通用表示

  - 阶段 2：微调（用显式数据）

    再用少量显式数据训练显式分支；共享层做 Stop Gradient（不更新）；

迁移本质：先学通用知识，再迁移到特定小数据任务，完全符合迁移学习流程。

4. 正则化效果：隐式任务防止目标任务过拟合（迁移带来的泛化提升）

  - 只用显式数据：模型越大越容易过拟合、RMSE 变差；
  - 加入隐式迁移：模型越大效果越好、泛化更强；


体现：源任务数据对目标任务起到正则化作用，是迁移学习带来的关键增益。



## 数据集

本文实验基于**Google 内部大规模应用推荐检索数据集**开展，该数据集为工业真实场景数据，未公开。数据集包含**数百万级候选 App**、**数十万条人工标注的显式相关性反馈**以及**数十亿条用户行为隐式反馈（点击、安装等）**。实验按 9:1 比例划分训练集与测试集，采用 RMSE 作为相关性预测的评价指标。



## 个人疑问

1、仅仅使用工业级数据集，没有使用公开数据集。效果不可复现。

2、相比于业界常用的召回评估指标Recall@K，Hit@K，NDCG等，仅仅使用RMSR指标不具有普遍性。



论文在**可复现性**与**评估合理性**上的两大短板，下面结合论文原文与工业视角，逐条回应并给出学术表述。

一、关于 “仅用工业私有数据集，效果不可复现”

**核心问题：无公开数据集、无开源代码，结论无法独立验证。**

原文事实

论文明确说明：

- 实验基于**Google 内部大规模 App 推荐检索系统**，**数据集完全私有、未公开**；
- 数据规模：**数百万候选、数十万显式标注、数十亿隐式日志**，均为内部业务数据；
- 无开源代码、无数据公开，第三方无法复现实验流程与结果。

学术缺陷

1. **缺乏可复现性**：论文结论依赖 Google 私有数据，外部研究者无法验证模型有效性，削弱结论普适性；
2. **数据分布未知**：私有数据的特征分布、标注规则、噪声水平不透明，**效果提升可能依赖特定业务数据特性，而非模型通用能力**；
3. **对比缺失**：无法在公开基准（如 MovieLens、Amazon、MSMARCO）上与主流召回模型（如双塔、NCF、DPR）横向对比，无法证明模型优势的通用性。

二、关于 “仅用 RMSE，未用 Recall@K、NDCG 等召回指标，评估不通用”

**核心问题：主任务是相关性回归，而非召回排序，指标选择脱离检索场景常规评估体系。**

原文事实

- 模型主任务：**显式相关性回归（MSE/RMSE）**，评估仅用**相关性预测 RMSE**；
- 未使用**Recall@K、Hit@K、NDCG、MAP**等检索 / 召回场景标准排序指标；
- 实验聚焦 “相关性预测精度”，而非 “召回结果排序质量”。

学术 + 工业缺陷

1. **指标与场景错位**：召回系统核心目标是**从海量候选中召回 Top-K 相关物品（排序能力）**，RMSE 仅衡量 “相关性分数拟合误差”，**无法反映召回排序效果**；
2. **缺乏排序有效性验证**：低 RMSE 不代表召回的 Top-K 物品更相关，可能出现 “分数准但排序差”，与工业召回核心诉求脱节；
3. **业界共识缺失**：工业与学术界召回任务均以**Recall@K、NDCG**为核心评估指标，仅用 RMSE 无法与主流工作横向对比，结论参考价值有限。

三、总结（可直接写入论文质疑 / 讨论部分）

该论文存在**可复现性不足**与**评估指标脱离场景**两大局限：其一，实验基于 Google 内部私有数据集，无公开数据与开源代码，第三方无法复现结果，结论普适性存疑；其二，模型聚焦相关性回归任务，仅采用 RMSE 作为评估指标，未使用 Recall@K、NDCG 等召回场景标准排序指标，无法验证模型在真实召回任务中的有效性，与工业界核心诉求存在偏差。
